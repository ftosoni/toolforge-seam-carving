# Toolforge Seam Carving Utility

An interactive, high-performance web-based **Seam Carving (Content-Aware Image Resizing)** utility designed for the Wikimedia Toolforge ecosystem. 
Features an interactive visualizer, brush painting for protection/removal constraints, and a dual-execution model supporting both pure client-side JavaScript processing and a multi-threaded C++ backend.

## ✨ Key Features

- **✂️ Content-Aware Resizing**: Shrink or enlarge images while preserving structural features (faces, text, objects) by removing lowest-gradient energy seams.
- **⚡ Dual-Resizing Engine**:
  - **Client-Side JavaScript**: Runs directly in the browser. Recommended for fast, interactive resizing of smaller/medium images.
  - **Multi-threaded C++ Backend**: Powered by an OpenMP-parallelized C++ submodule for static image calculations.
- **🎨 Constraint Brush Painting**:
  - 🟢 **Protect (Green)**: Assigns positive energy weights to prevent critical regions from being carved.
  - 🔴 **Remove (Red)**: Assigns negative energy weights to force the removal of unwanted objects.
  - 🧹 **Local Reset**: Easily erase constraints in specific areas.
- **🔮 Real-time Interactive Visualizer**: Animate the carving process step-by-step and highlight active seams in real-time.
- **🧠 Advanced Energy Formulas**: Supports both classic **Backward Energy** (gradient Sobel filters) and Rubinstein's 2008 **Forward Energy** (minimizing introduced energy/distortion).
- **🌍 Massive Localization Footprint**: Fully localized UI supporting **17 languages** (including Assamese, Bengali, Hindi, Santali, Odia, Malayalam, Italian, French, Urdu, and more).
- **🌊 REST API Endpoint**: Exposes a POST endpoint at `/api/carve` wrapped in FastAPI with auto-generated Swagger UI and real-time streaming progress logs.

## 🚀 Running Locally

### 1. Compile C++ Submodule (Optional)
To run the server-side API backend locally, you must first compile the C++ binary:
```bash
cd seam-carving
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
cd ..
```
The compiled binary will be placed inside `seam-carving/build/` and automatically detected by `app.py`.

### 2. Start the FastAPI Server
Install python dependencies and start Uvicorn:
```bash
pip install -r requirements.txt
uvicorn app:app --reload
```
- Open `http://localhost:8000` to access the interactive web interface.
- Open `http://localhost:8000/docs` to view and test the autogenerated Swagger API documentation.

## 🚀 Deployment (Toolforge)

Deploying to Wikimedia Toolforge is fully automated via Cloud Native Buildpacks (CNB) and the Toolforge Build Service.

### 1. Setup the Web Service
Log into Toolforge and switch to your project's account:
```bash
ssh <your-developer-name>@login.toolforge.org
become seam-carving
```

### 2. Build the Container
Start a new build from your GitHub repository:
```bash
# Clean existing builds
toolforge webservice buildservice stop --mount=all
toolforge build clean -y

# Start the build pipeline
toolforge build start https://github.com/ftosoni/toolforge-seam-carving
```
The build service will parse `project.toml`, install `cmake`, `g++`, `make`, and `build-essential` libraries, and compile the container.

### 3. Launch and Monitor
Start the web service and tail the logs. The `start.sh` entrypoint script will automatically patch the buildpack's linker scripts and compile the C++ binary:
```bash
# Start webservice
toolforge webservice buildservice start --mount=all

# Tail startup and execution logs
toolforge webservice logs -f
```
